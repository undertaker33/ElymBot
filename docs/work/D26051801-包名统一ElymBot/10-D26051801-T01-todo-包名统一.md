# D26051801-T01 Todo 包名统一

## 任务边界

执行项目自有历史命名到 `ElymBot` 的统一迁移。

## 允许修改

- Gradle、Manifest、source set package、Kotlin/Java 文件名和目录。
- 当前源码、测试、资源、脚本、示例数据中的项目自有旧名。
- 当前治理文档、README、context 和本任务包。
- Room schema 目录和 JSON 中的项目自有 package 名。

## 禁止修改

- Git commit / branch / tag / push / PR。
- 与改名无关的架构重构、业务逻辑改造、依赖升级。
- 本轮开始前已有未跟踪生成物清理。
- 第三方上游事实或外部兼容协议，除非实现证明它们是本项目自有旧名。

## 执行步骤

1. 建立旧名命中清单，区分项目自有命名与外部保留命名。
2. 批量迁移 package 目录、package 声明、import、Gradle namespace/applicationId、manifest class、测试路径。
3. 迁移项目自有类名、函数名、文件名和脚本变量。
4. 更新当前文档层和任务反馈。
5. 运行 UTF-8 guard、architecture/build/test 验证。

## 验证

优先执行：

```powershell
.\gradlew.bat architectureCheck --console=plain --no-daemon --stacktrace
.\gradlew.bat :build-logic:check --console=plain --no-daemon --stacktrace
.\gradlew.bat :app:testDebugUnitTest --console=plain --no-daemon --stacktrace
.\gradlew.bat clean assembleDebug --console=plain --no-daemon --stacktrace
```

补充扫描：

```powershell
rg --hidden --no-ignore --glob '!**/.git/**' --glob '!**/.worktrees/**' --glob '!**/build/**' --glob '!**/.gradle/**' --glob '!**/bin/**' --glob '!logs/**' --glob '!artifacts/apk/**' --glob '!**/*.zip' "<legacy package / class / deep-link pattern>"
```

## 交付反馈要求

反馈文件记录：

- 修改文件范围。
- 迁移策略。
- 保留的外部兼容旧名及原因。
- 验证命令、结果、warning / exception 计数。
- 未验证项、风险和回滚方式。
