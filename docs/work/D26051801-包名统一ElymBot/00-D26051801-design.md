# D26051801 包名统一 ElymBot Design

## 背景

项目已经统一定名为 `ElymBot`，但当前源码、Gradle、Android package、测试包路径、文档和部分项目自有标识仍保留历史旧名。

本任务作为正式开发任务处理，目标是把项目自有旧名称统一到 `ElymBot` 口径，并通过构建与架构入口验证改名没有破坏生产主线。

## 目标

- Android `namespace`、`applicationId`、Kotlin/Java package 迁移到 `com.elymbot.android`。
- 项目自有类名、函数名、文件名、测试名、资源名和脚本变量中的旧项目名迁移到 `ElymBot` / `elymbot`。
- 当前文档入口、context、README、任务文档中描述项目旧名的位置同步到新口径。
- 保持 Gradle、Hilt、Room schema、architecture tests、runtime scripts 和插件运行时可编译。

## 非目标

- 不改变业务功能、UI 流程、模型能力、插件能力或运行时策略。
- 不执行 Git 写入。
- 不清理本轮开始前已有的未跟踪生成物：`architecture-tests/bin/`、`build-logic/bin/`、`logs/`。
- 不盲目改写外部上游项目、第三方仓库、兼容协议或公开插件协议中的 `AstrBot` 事实；如果这些名称属于外部兼容契约，必须在实现反馈里列为保留项。

## 影响范围

- Root Gradle：`settings.gradle.kts`、`build.gradle.kts`。
- App 壳层：`:app`、`:app-integration`、Manifest、Application、Activity、navigation root。
- 全部 source set：`app/`、`app-integration/`、`core/`、`download/`、`feature/`、`architecture-tests/`。
- 资源和脚本：`app/src/main/res/**`、`app/src/main/assets/runtime/scripts/**`、测试资源。
- 文档：`AGENTS.md`、`README.md`、`docs/**/*.md` 当前文档层与本任务包。

## 验收标准

- legacy package / class / deep-link 精确扫描在当前源文件和文档范围内无项目自有命中。
- 项目自有 `ElymBotApplication`、`ElymBotApp`、`ElymBotAppScaffoldParts` 等标识改为 `ElymBot*`。
- `.\gradlew.bat architectureCheck --console=plain --no-daemon --stacktrace` 通过。
- `.\gradlew.bat :build-logic:check --console=plain --no-daemon --stacktrace` 通过。
- `.\gradlew.bat :app:testDebugUnitTest --console=plain --no-daemon --stacktrace` 通过。
- `.\gradlew.bat clean assembleDebug --console=plain --no-daemon --stacktrace` 通过，且日志扫描无 warning / deprecated / exception。
- Governed Markdown 写入前后通过 `uth-utf8-guard`。

## 风险

- 大规模 package 迁移会影响 Hilt/KSP 生成路径、Room schema 路径、architecture tests 的源码路径和 allowlist。
- Room schema 目录改名可能影响历史 schema 识别，需要验证 `room.schemaLocation` 与现有 schema JSON 的 package 名一致性。
- 插件兼容协议如果直接改名，可能破坏已有插件生命周期 hook 或示例插件元数据；实现阶段需要区分项目自有标识与外部契约。
