# D26051801-T01 Feedback 包名统一

## 结论

本轮作为 `uth-dev` / `formal-dev` 正式任务完成项目自有 Android 旧命名到 `ElymBot` 口径的统一迁移。

迁移后主包、Gradle `namespace` / `applicationId`、Kotlin package、Room schema 目录、Manifest、Application、Compose app shell、测试包路径、脚本变量、示例数据和当前治理文档均对齐到 `ElymBot` / `com.elymbot.android`。

## 修改范围

- Root Gradle 与模块 Gradle：项目名、`namespace`、`applicationId`、测试过滤和模块源码包引用。
- App 壳层与集成层：Manifest、Application、Activity、navigation root、Hilt wiring、startup chain、app chrome。
- 全部源码 source set：`app/`、`app-integration/`、`core/`、`download/`、`feature/`、`architecture-tests/`。
- Room schema：schema 目录和 JSON 中的数据库类、实体 bundle、identity hash 相关 owner 字段按新 package 重新落位。
- Runtime assets 与脚本：容器、NapCat、TTS、下载目录、环境变量和临时路径改为新前缀。
- 插件平台：项目自有 lifecycle hook 和示例包名改为 `on_elymbot_loaded`、`OnElymBotLoaded`、`onElymBotLoaded` 口径。
- 文档：`AGENTS.md`、`README.md`、`docs/` 当前上下文、归档文档和本任务包同步当前命名。

## 迁移策略

1. 先按项目自有旧名集合做批量内容替换，覆盖 package、类名、资源名、脚本变量、深链和示例数据。
2. 再迁移物理路径和文件名，使 `com/elymbot/android` 与 Kotlin package 声明一致。
3. 对构建失败做最小修复：首次架构入口失败来自旧 generated output，清理构建目录后重新验证通过。
4. 对 README / project overview / 任务文档中被机械替换后语义不自然的句子做人工修正。
5. 通过精确旧名扫描确认项目自有旧标识已清空；通过广义扫描确认剩余命中只属于上游 `AstrBot` 事实或历史归档语境。

## 保留项

- README 和项目概览中对上游 `AstrBot` 官方项目的归属说明保留，这是外部事实，不属于 Android 项目自有包名。
- `docs/archive/**` 中部分历史语境允许保留上游或历史说明；它们不作为当前代码事实源。
- `keystore.properties` 是 Git 忽略的本地发布签名配置，当前指向工作区外的既有 release keystore；本轮不做签名凭据文件名或 alias 轮换，避免破坏 release 签名。
- 本轮开始前已有未跟踪生成物 `architecture-tests/bin/`、`build-logic/bin/`、`logs/` 未清理。
- 未执行 Git commit / branch / tag / push / PR。

## 风险与后续

- 这是大规模路径迁移，当前 Git 状态会显示旧路径删除与新路径未跟踪成对出现；进入 `uth-git` 前应按 rename diff 审核并一次性提交。
- 旧 package 变更会影响 Android 安装身份和已有本地数据隔离；如需面向已安装用户平滑迁移，需要另开数据迁移 / 发布策略任务。
- 插件 hook 已统一到新项目名；若后续需要兼容旧插件 hook，应另开兼容层任务，不在本轮直接恢复旧命名。
