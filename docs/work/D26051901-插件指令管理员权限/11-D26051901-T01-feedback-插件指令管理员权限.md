# D26051901-T01 插件指令管理员权限 Feedback

更新时间：2026-05-20 00:00 +08:00

## 原始需求

插件指令需要接入权限系统：在配置页中加一个开关，控制是否仅管理员可触发指令。

## 实现摘要

- `ConfigProfile` 新增 `pluginCommandsAdminOnlyEnabled`，默认 `false`。
- 配置详情页管理员设置区新增“仅管理员可触发斜杠指令”开关，并纳入保存草稿。
- QQ 斜杠指令入口新增 `QqSlashCommandPermissionPolicy`，开关开启且发送者非管理员时拦截并消费内置 bot command 与插件 command fallback。
- Room database 升到 v23，新增 22 -> 23 迁移和双份 schema export。
- 备份导入导出、legacy config import、runtime context、plugin runtime model adapter 同步该字段。
- 增加权限策略、schema 合同和 config mapper 测试。

## Debug 追加：内置 `/help` 绕过权限

用户在 2026-05-20 反馈非管理员仍可使用 `/help`，日志显示命中 `Bot command handled via router: /help`。

根因：首轮实现只在 `QqPluginDispatchService.handlePluginCommand(...)` 接入权限策略；QQ 消息链路会先解析 `BotCommandRouter.supports(parsedBotCommand.name)`，内置 `/help`、`/stop`、`/op` 等命令直接进入 `QqBotCommandRuntimeService.handle(...)`，不会落到插件 command fallback。

修复：将权限策略提升为 `QqSlashCommandPermissionPolicy`，同时接入 `QqBotCommandRuntimeService` 和 `QqPluginDispatchService`；配置页文案从“插件指令”调整为“斜杠指令”。

新增回归测试：


Red：修复前失败，实际返回 `/help` 帮助文本。

Green：修复后通过，非管理员 `/help` 返回权限拒绝文本，不再返回帮助文本。
