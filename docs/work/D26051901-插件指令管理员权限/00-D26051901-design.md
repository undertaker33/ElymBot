# D26051901 插件指令管理员权限 Design

更新时间：2026-05-20 00:00 +08:00

## 背景

用户要求插件指令接入权限系统：在配置页新增开关，控制是否仅管理员可触发指令。运行验证后明确该开关需要覆盖 QQ 侧内置斜杠指令和插件斜杠指令，而不是只覆盖 plugin dispatch fallback。

本任务影响配置模型、Room 持久化、配置页 UI、QQ 插件指令入口、runtime context 映射、备份恢复兼容和对应测试。

## 目标

- `ConfigProfile` 增加插件指令管理员限制开关，默认关闭。
- 配置详情页在管理员设置区提供该开关，并纳入保存草稿。
- QQ 侧斜杠指令在开关开启时只允许管理员 UID 触发，包括内置 bot command 和插件 command fallback。
- 新字段进入 Room schema、迁移、schema reset、备份导入导出、legacy import 与 runtime snapshot 映射。
- 用单测覆盖权限策略、Room schema 迁移合同和 config mapper 行为。

## 非目标

- 不改变 wake word、白名单、限流等既有权限策略。
- 不改变插件自身 manifest / trigger schema。
- 不新增 Git 提交、分支、tag 或 PR。
- 不扩展到 Cron 或 App Chat 插件消息入口。

## 方案

1. 在 config domain model 和 Room entity 增加 `pluginCommandsAdminOnlyEnabled`。
2. 将该字段贯通 config mapper、legacy import、backup manifest adapter、backup repository、runtime context adapter 和 plugin runtime model adapter。
3. Room database 版本从 22 升到 23，新增 `migration22To23`，同步 schema export 与 reset SQL。
4. 在配置详情页 `PlatformSettingsSection` 增加管理员开关。
5. 在 QQ bot command 和 plugin command dispatch 前加入 `QqSlashCommandPermissionPolicy` 判定；开关开启且发送者不是管理员时消费该指令，并按 `replyWhenPermissionDenied` 决定是否回复权限提示。
6. 增加聚焦单测和 schema 合同断言。

## 验收标准

- 非管理员在开关开启时不能触发 QQ `/help` 等内置斜杠指令，也不能触发 QQ 插件斜杠指令。
- 管理员在开关开启时仍可触发。
- 开关关闭时保持原有插件指令行为。
- Room v23 schema 包含新字段，迁移链包含 22 -> 23。
- `clean architectureCheck assembleDebug` 和 `:app:testDebugUnitTest` fresh 通过，日志扫描 warning / deprecated / exception 为 0。
