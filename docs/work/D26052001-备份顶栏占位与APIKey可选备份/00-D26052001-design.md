# D26052001 备份顶栏占位与 API-Key 可选备份设计

## 目标

本任务实现三个已确认需求：

- 备份三级页内容整体下移，避免被全局 secondary top bar 遮挡。
- 全局顶栏在备份三级页显示对应模块标题，而不是统一显示“数据备份”。
- 完整备份和模型备份在点击“立即备份”后弹出确认弹窗，提供默认关闭的“是否备份 API-Key”开关；只有用户打开时才把 Provider `apiKey` 写入备份。

## 范围

允许修改：

- 备份 UI：`feature/settings/presentation/src/main/java/com/elymbot/android/ui/settings/**`
- settings backup API：`feature/settings/api/src/main/java/com/elymbot/android/feature/settings/api/SettingsBackupPort.kt`
- app settings adapter：`app/src/main/java/com/elymbot/android/feature/settings/AppSettingsBackupPortAdapter.kt`
- backup repository：`app/src/main/java/com/elymbot/android/core/db/backup/AppBackupRepository.kt`
- app 顶栏路由与字符串装配：`app/src/main/java/com/elymbot/android/ui/app/**`, `core/ui/src/main/java/com/elymbot/android/ui/app/TopBarRegistration.kt`
- app 字符串资源：`app/src/main/res/values*/strings.xml`
- 直接相关备份测试。

禁止修改：

- Git 写入、分支、提交、tag、PR。
- 数据库 schema / migration。
- 与本任务无关的 D26051901 插件管理员权限改动。
- backup 覆盖范围扩张到 Resource Center、Cron 或 Plugin。

## 设计

### 顶部占位

`SettingsSubPageScaffold` 不再绘制内部 back/title 顶栏，改为复用当前全局 secondary top bar 占位体系：

- `topBar = { SecondaryTopBarPlaceholder() }`
- `contentWindowInsets = WindowInsets.safeDrawing`
- `containerColor = MonochromeUi.pageBackground`

页面内容继续通过 `innerPadding` 进入 `LazyColumn`。这只解决遮挡，不制造第二条顶栏。

### 全局顶栏标题

`SecondaryTopBarStrings` 增加 backup 子模块标题字段，`ElymBotApp` 从现有字符串资源读取：

- `backup_module_bots_title`
- `backup_module_models_title`
- `backup_module_personas_title`
- `backup_module_conversations_title`
- `backup_module_configs_title`
- `backup_module_tts_title`
- `backup_module_full_title`

`GlobalTopBar.fallbackSecondaryTopBarSpecForRoute(...)` 对 backup route 做逐项映射：

- `backup-hub` -> `dataBackup`
- `backup/bots` -> `backupBots`
- `backup/models` -> `backupModels`
- `backup/personas` -> `backupPersonas`
- `backup/conversations` -> `backupConversations`
- `backup/configs` -> `backupConfigs`
- `backup/tts` -> `backupTts`
- `backup/full` -> `backupFull`

### API-Key 可选备份

当前 `providerToJson(...)` 无条件写出 `apiKey`。新语义改为默认不写出真实 API-Key：

- `includeProviderApiKeys = false` 时，Provider 记录中的 `apiKey` 写为空字符串。
- `includeProviderApiKeys = true` 时，Provider 记录中的 `apiKey` 写出原值。
- 旧备份导入继续按 `JSONObject.toProviderProfile()` 读取 `apiKey`，保持兼容。

参数从 UI 传递到 repository：

1. `BackupScreen` 在完整备份和模型备份点击“立即备份”后弹出确认弹窗。
2. 弹窗包含一个默认关闭的开关“是否备份 API-Key”，确认后调用 ViewModel。
3. `SettingsBackupViewModel` 透传 `includeProviderApiKeys`。
4. `SettingsBackupPort` / `AppSettingsBackupPortAdapter` / `AppBackupService` / `AppBackupRepository` 透传参数。
5. `buildManifest(...)` 和 `buildModuleManifest(...)` 用参数控制 `providerToJson(...)`。

## 验收

- `backup/full` 顶栏显示“完整备份”，`backup/models` 显示“模型备份”，其他备份三级页显示对应模块标题。
- 所有备份三级页第一张卡片位于全局顶栏下方，不被遮挡。
- 完整备份和模型备份点击“立即备份”先弹确认弹窗，开关默认关闭。
- 默认备份生成的 Provider JSON 中 `apiKey` 为空。
- 打开“是否备份 API-Key”后备份生成的 Provider JSON 中 `apiKey` 保留原值。
- 旧备份导入不破坏。
