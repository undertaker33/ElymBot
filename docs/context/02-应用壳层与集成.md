## 模块职责

`:app` 当前负责 Android 应用壳层：

- Manifest、Application、Activity、主题、资源和应用级权限声明。
- `@HiltAndroidApp` / `@AndroidEntryPoint` 入口。
- WorkManager `Configuration.Provider` 和 `HiltWorkerFactory` 接入。
- `AppBootstrapper -> AppStartupRunner -> di/startup/*` 启动顺序入口。
- Compose app shell、导航根、top bar、drawer、bottom navigation 和模块 presentation screen 组合。
- 平台薄胶水：通知权限、runtime bridge auto-start、插件 deep link 确认入口。

`:app-integration` 当前负责跨模块 Hilt wiring：

- 暴露 core、download API、feature API 给 app 层编译可见。
- 内部以 `implementation(project(...))` 接入 feature data / runtime / impl 和 download impl，作为生产 wiring 边界。
- 提供 adapter、binding、contributor、module 等接线型文件。
- 不拥有 feature 业务逻辑、UI、长运行 runtime loop 或进程执行逻辑。

## 非职责

- `:app` 不作为 feature data / runtime / impl 的生产 owner。
- `:app` 的 `testImplementation(project(...))` 依赖只作为测试 classpath 处理，不代表生产主线依赖。
- `:app-integration` 不应通过 `api(project(...))` 暴露 feature data / runtime / impl 给 `:app`。
- `:app-integration` 不应依赖 feature presentation、Compose UI 或 Android application plugin。
- 生产依赖注入唯一合法方法仍是 Hilt；不得恢复手写 app container、service locator、global registry 或 static install/configure callback。

## 当前入口事实

`app/build.gradle.kts` 的当前事实：

- `namespace = "com.elymbot.android"`
- `applicationId = "com.elymbot.android"`
- `targetSdk = 36`
- `versionCode = 73`
- `versionName = "0.9.1"`
- 生产依赖只直接接入 `:core:ui`、`:app-integration` 和 feature `:presentation` 模块。
- `prepareFilteredAssets` 把运行时大资源排除后写入 `generated/filtered-assets/main`，再作为 main assets source dir。

`app-integration/build.gradle.kts` 的当前事实：

- `namespace = "com.elymbot.android.app.integration"`
- `api(project(...))` 主要暴露 core runtime / backup / db / logging / network、download API 和 feature API。
- `implementation(project(...))` 接入 download impl 以及 feature data / runtime / impl，作为 app 不直接依赖实现模块的 wiring 层。
- 当前未看到 feature presentation 依赖。

Manifest 与网络配置当前事实：

- Application 为 `.ElymBotApplication`。
- `android:allowBackup="false"`。
- `android:usesCleartextTraffic="false"`。
- `android:networkSecurityConfig="@xml/network_security_config"`。
- WorkManager 默认 initializer 被移除，由 Hilt worker factory 配置接管。
- `.core.runtime.container.ContainerBridgeService` 是非导出的 foreground `dataSync` service。
- `.MainActivity` 是 launcher activity，并接收 `elymbot://plugin/repository` 与 `elymbot://plugin/install` deep link。
- `network_security_config.xml` 只允许 `127.0.0.1` 和 `localhost` cleartext。

Application / Activity 当前事实：

- `ElymBotApplication` 使用 `@HiltAndroidApp`。
- `ElymBotApplication.onCreate()` 调用 `appBootstrapper.bootstrap()`。
- WorkManager 配置使用注入的 `HiltWorkerFactory`。
- `MainActivity` 使用 `@AndroidEntryPoint`。
- `MainActivity` 注入 `PluginCatalogRuntimePort`、`ContainerBridgeStatePort`、`RuntimeBridgeController` 和 `RuntimeLogger`。
- 插件 deep link 先写入 `pendingPluginDeepLinkRequest`，确认后才调用 `pluginCatalogRuntimePort.handleInstallIntent(...)`。
- runtime bridge auto-start 以 `bridgeStatePort.config` 和 `bridgeStatePort.runtimeState` 判断。

启动链当前事实：

1. `BootstrapPrerequisitesStartupChain`
   - 设置默认 uncaught exception handler。
   - 调用 `downloadManagerBootstrapPort.ensureReady()`。
   - 调用 `qqLoginStateBootstrapper.ensureReady()`。
   - 调用 `qqStartupPort.initialize()`。
2. `RepositoryInitializationStartupChain`
   - 当前 `run()` 为空实现。
3. `ReferenceGuardStartupChain`
   - 当前 `run() = Unit`。
4. `PluginRuntimeObservationStartupChain`
   - 调用 `pluginRuntimeObservationPort.startObserving()`。
5. `RuntimeLaunchStartupChain`
   - 调用 `qqStartupPort.start()`。
   - 调用 `containerRuntimeInstaller.warmUpAsync(appScope)`。
   - 调用 `cronRuntimeReconciliationPort.reconcileAsync(appScope)`。
   - 记录 `App started`。

导航壳层当前事实：

- `ElymBotApp()` 位于 `app/src/main/java/com/elymbot/android/ui/app/ElymBotApp.kt`。
- `ElymBotApp()` 使用 `rememberNavController()` 和多个 `hiltViewModel()` 装载 app shell 所需 ViewModel。
- `AppDestination` 是 sealed route 定义，覆盖 bots、plugins、chat、config、logs、me、resource、QQ、settings、assets、backup、models、runtime、cron jobs 等入口。
- `ElymBotAppNavGraph` 的 `startDestination` 是 `AppDestination.Chat.route`。
- `ElymBotAppScaffoldParts.kt` 组合 feature presentation screen；这是 app shell composition，不表示 app 接管 feature 实现 owner。
