## 模块职责

`verification-build-governance` 负责记录和维护以下事实入口：

- 根 Gradle 验证任务和 module group 任务。
- `architectureCheck` 的组成。
- 独立 `:architecture-tests:test` 的源码与资源接入方式。
- `build-logic` Kotlin/JVM convention plugin 的职责。
- architecture allowlist 文件和债务统计口径。
- 文档里应该引用哪些验证入口，以及哪些验证结果只是历史证据。

## 非职责

本模块不负责：

- 修改 Gradle、build logic、测试或源码。
- 运行 Gradle 构建、单测或 instrumentation test。
- 证明当前工作树可编译。
- 判定业务模块自身实现是否正确。
- 维护 release changelog 或 Git closure。

如果后续发现验证入口本身损坏，应路由到 `uth-debug` 或 `uth-dev`；如果需要提交、推送或发布，应路由到 `uth-git`。

## 当前验证入口

根验证入口来自 `build.gradle.kts`：

```powershell
.\gradlew.bat architectureCheck --console=plain --no-daemon --stacktrace
.\gradlew.bat :architecture-tests:test --console=plain --no-daemon --stacktrace
.\gradlew.bat :build-logic:check --console=plain --no-daemon --stacktrace
.\gradlew.bat :app:architectureDebugUnitTest --console=plain --no-daemon --stacktrace
.\gradlew.bat :app:testDebugUnitTest --console=plain --no-daemon --stacktrace
.\gradlew.bat :app:assembleDebug --console=plain --no-daemon --stacktrace
.\gradlew.bat clean assembleDebug --console=plain --no-daemon --stacktrace
```

`architectureCheck` 当前依赖：

- `architectureSourceRootsReport`
- `architectureDebtReport`
- `:architecture-tests:test`
- `:app:architectureDebugUnitTest`

其中 `architectureSourceRootsReport` 会按 `architectureMainSourceRoots` 检查源根；`architectureDebtReport` 会读取：

- `app/src/test/resources/architecture/global-singleton-allowlist.txt`
- `app/src/test/resources/architecture/static-repository-usage-allowlist.txt`

## architecture-tests 边界

`:architecture-tests` 是独立 JVM 测试入口：

- 使用 `elymbot.kotlin.jvm` convention plugin。
- 依赖 `junit:junit:4.13.2`。
- `test` sourceSet 指向 `app/src/test/java/com/elymbot/android/architecture`。
- `test` resources 指向 `app/src/test/resources`。
- Test working directory 固定为根项目目录。

这表示 architecture contract 的源码真源仍在 `app/src/test/java/com/elymbot/android/architecture`，`:architecture-tests` 是独立执行壳层，不是重复源码目录。

当前跟踪到 44 个 architecture contract test 文件，以及 9 个 architecture resource / allowlist 文件。

## build-logic 边界

`settings.gradle.kts` 通过 `includeBuild("build-logic")` 接入本地构建逻辑。

当前构建链版本事实：

- Android Gradle Plugin：`9.2.0`
- Gradle Wrapper：`9.4.1`
- Kotlin JVM / Compose plugin：`2.2.10`
- KSP：`2.3.8`
- Hilt：`2.59.2`
- `compileSdk = 36`，`compileSdkMinor = 1`，`minSdk = 29`

`build-logic` 当前只注册一个 convention plugin：

- plugin id：`elymbot.kotlin.jvm`
- implementation class：`ElymBotKotlinJvmConventionPlugin`
- 行为：应用 `org.jetbrains.kotlin.jvm`、设置 Kotlin `jvmTarget = "17"`、为 `Test` 任务启用 JUnit。

`build-logic/gradle.properties` 当前关闭 Kotlin incremental，并使用 in-process compiler execution。修改该区域时，不能只跑 app 编译；需要把 `:build-logic:check` 当作验证入口。

CI 当前在 tooling 阶段加入 `Verify generated artifacts are not tracked`，用 `git ls-files 'artifacts/**'` 阻断生成产物被提交；`.gitignore` 同步排除 `artifacts/`。

## module group 任务

根 `build.gradle.kts` 维护 `moduleBuildGroups`，并派生：

- `<taskPrefix>Build`
- `<taskPrefix>Check`
- `allModuleGroupsBuild`
- `allModuleGroupsCheck`

`*Build` 会在 projects evaluated 后依赖每个模块的 `assembleDebug` 或 `assemble`；`*Check` 会依赖对应模块的 `check` 并先执行 `*Build`。

当前 module group 包括：

- `moduleCoreFoundation`
- `moduleCoreRuntime`
- `moduleDownload`
- `moduleBot`
- `moduleChat`
- `moduleConfig`
- `moduleConversation`
- `moduleCron`
- `modulePersona`
- `modulePlugin`
- `moduleProvider`
- `moduleQq`
- `moduleResource`
- `moduleSettings`
- `moduleVoiceAsset`
- `moduleAppShell`

如果新增 Gradle 子项目，需要同时核对：

- `settings.gradle.kts`
- `architectureMainSourceRoots`
- `moduleBuildGroups`
- 相关 architecture contract
- `docs/module-build-guide.md`
