## 模块职责

本模块当前覆盖：

- `:feature:conversation:api`：conversation/session/message/attachment model 与 `ConversationRepositoryPort`。
- `:feature:conversation:data`：Room-backed conversation store、legacy JSON import、session import/export、title sync、migration adapter 和 repository port adapter。
- `:feature:chat:api`：App Chat runtime port、发送用例、消息更新节流策略和 use case factory。
- `:feature:chat:presentation`：Chat screen、ChatViewModel、ConversationViewModel、session drawer/input 组件、session controller 和 presentation-level runtime binding contract。
- `:feature:chat:runtime`：App Chat runtime service、provider invocation、prepared reply、plugin command ingress、bot command router/resources 和 runtime service factories。
- `app-integration` 的 chat adapter：把 feature data store、runtime factory、plugin runtime 和 presentation binding contract 接到生产 Hilt graph。
- app root 的 `DefaultChatViewModelRuntimeBindings`：当前生产 Hilt binding 实现，仍是 app/app-integration 过渡层的一部分。

## 非职责

- QQ / NapCat / OneBot 平台消息接收、reverse WS、QQ bot command service 不在本模块完成态；后续 `qq-napcat-onebot` 单独治理。
- Plugin 平台 runtime、host capability、tool source 注册和 plugin config state 不在本模块完成态；这里只确认 App Chat 如何调用 plugin runtime。
- Cron runtime closure 与 scheduled message delivery 不在本模块完成态；这里只确认 App Chat runtime factory 当前注入 `ScheduledTaskIntentFallbackResponder`。
- Runtime context、LLM/core runtime 基础设施已在 `core-runtime` 确认；本模块只记录 App Chat 对这些 port 的消费方式。
- Conversation backup UI、settings backup adapter 和完整备份入口不在本模块完成态；后续 `resource-settings-backup` 单独治理。

## Conversation 数据真源

当前 conversation model 物理位于 `feature/conversation/api/src/main/java/com/elymbot/android/feature/chat/model/**`，但 package 是 `com.elymbot.android.model.chat`。不要只按物理路径推断 package。

当前核心模型：

- `ConversationSession`
- `ConversationMessage`
- `ConversationAttachment`
- `ConversationToolCall`
- `MessageSessionRef`
- `MessageType`

`defaultSessionRefFor(sessionId)` 当前语义：

- `qq-*-private-*` 映射为 `platformId = "qq"`、`MessageType.FriendMessage`、`originSessionId = "friend:<userId>"`。
- `qq-*-group-*` 映射为 `platformId = "qq"`、`MessageType.GroupMessage`，并保留 isolated user 形式。
- 其他 session 默认为 `platformId = "app"`、`MessageType.OtherMessage`、`originSessionId = sessionId`。

`FeatureConversationRepositoryStore` 当前事实：

- 是 `@Singleton`。
- 使用 `ConversationAggregateDao` 作为 Room persistence 真源。
- 使用 `@Named("legacyConversationStorageFile")` 读取旧 JSON 会话并迁入 Room。
- 默认会话 ID 是 `chat-main`，默认标题源码中以 Unicode escape 表示为“新对话”。
- 初始化时如果 DB 为空且没有 legacy import，会写入默认 session。
- `sessions` 是 `StateFlow<List<ConversationSession>>`。
- `appendMessage`、`updateMessage`、`replaceMessages`、`renameSession`、`deleteSession`、`toggleSessionPinned`、`updateSessionBindings`、`updateSessionServiceFlags` 均在 store 中更新内存状态并持久化。
- `updateMessage` 使用 300ms debounce 持久化，适配 streaming assistant delta。
- durable restore/import 会在 Room 写入失败时回滚内存状态并重新抛出错误。
- import dedup key 对 QQ session 使用 platform、bot、message type 与 origin，app session 使用本地 session id。

生产 port：

- contract：`feature/conversation/api/src/main/java/com/elymbot/android/feature/conversation/domain/ConversationRepositoryPort.kt`
- adapter：`feature/conversation/data/src/main/java/com/elymbot/android/feature/conversation/data/FeatureConversationRepositoryPortAdapter.kt`
- Hilt binding：`app-integration/src/main/java/com/elymbot/android/di/hilt/RepositoryPortModule.kt`

`RepositoryPortSourceContractTest` 当前要求 `FeatureConversationRepositoryPortAdapter` 存在、legacy adapter 不再作为生产入口；旧 `LegacyConversationRepositoryAdapter` 名称不应写成当前事实。

## App Chat 发送链路

当前发送主链：

1. `ChatViewModel.sendMessage(...)` 处理 UI 输入、附件、当前 session、provider、bot、persona、STT/TTS 开关和 plugin ingress。
2. `AppChatSendHandler` 把 presentation request 映射到 domain use case。
3. `SendAppMessageUseCase` 写入 user message，执行 `beforeRuntime`，创建/更新 assistant message，并把 `AppChatRuntimeEvent` 映射为发送事件。
4. `AppChatRuntimeService` 通过 `RuntimeContextResolverPort.resolve(...)` 构造 runtime context，再通过 `RuntimeLlmOrchestratorPort.dispatchLlm(...)` 调 plugin v2 LLM pipeline。
5. `AppChatProviderInvocationService` 把 plugin pipeline provider request 映射回 App Chat provider 调用。
6. `AppChatPreparedReplyService` 处理模型 reply、附件映射和 TTS/voice reply attachment。
7. `AppChatPluginCommandService` 处理 App Chat plugin command / message ingress。

当前 `ChatViewModel` 仍位于 `feature/chat/presentation/.../ChatViewModel.kt`，但 package 是 `com.elymbot.android.ui.viewmodel`。`ConversationViewModel`、`ChatViewModelRuntimeBindings` 和 `ChatSessionController` 也保持该 package；路径与 package 需要双重确认。

`ChatViewModel` 当前额外约束：

- App Chat 发送如果当前选中 QQ session，会自动创建新的 app session，避免把 app 输入写入 QQ session。
- `handleSessionCommand(...)` 先让 unsupported slash command 走 plugin command service；supported command 走 `BotCommandRouter`。
- `BotCommandRouter` 支持 session/provider/persona/STT/TTS/LLM 等命令，并通过 context 回调写回 session/config/bot/provider。
- 音频附件会按当前 config 与 session flag 尝试 STT；TTS 触发来自 config always-on 或输入 `~` 后缀。
- `withSessionLock(sessionId)` 通过 `SessionLockCoordinator` 串行化同一会话的发送流程。
