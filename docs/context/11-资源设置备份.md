## 模块职责

本模块当前覆盖：

- `:feature:resource:api`：Resource Center model、resource / projection port、compatibility snapshot use case。
- `:feature:resource:data`：Room-backed Resource Center store、legacy config MCP / skill projection seed、`FeatureResourceCenterPortAdapter`。
- `:feature:resource:presentation`：Resource Center 页面、分类子页、Hilt ViewModel、presentation controller。
- `:feature:settings:api`：Settings 侧 backup UI 所需的 `SettingsBackupPort`、settings backup DTO、backup data ports。
- `:feature:settings:presentation`：Me 入口、Settings hub、Backup hub、module backup / full backup / conversation backup UI。
- `core/backup`：backup participant metadata contract 和 supported / planned coverage registry。
- `app/src/main/java/com/astrbot/android/core/db/backup`：当前 App backup / conversation backup 的具体 repository 与 injectable service 实现。
- `core/logging`：runtime log store、logger、cleanup settings 和 maintenance service。
- `core/runtime-cache`：可安全清理的 runtime cache target 与 `RuntimeCacheMaintenancePort`。
- `app-integration` wiring：通过 Hilt 绑定 Resource Center port、backup data ports、backup participant registry、runtime logging 和 runtime cache maintenance。
- `app` shell：settings hub、runtime config screen、logs screen、runtime cache cleanup card、backup port adapter 和导航 route 接入。

## 非职责

- Cron jobs、Cron execution records 已在第 10 模块确认；本模块只记录 backup 当前仍未覆盖它们。
- Plugin runtime、plugin config/state、ToolSource 和 plugin logs 已在第 9 模块确认；本模块只记录 backup participant 对 plugin data 的覆盖状态为 `PLANNED`。
- STT/TTS、voice clone asset data owner、TTS asset backup 细节和 audio runtime 仍留给第 12 模块 `voiceasset-audio`。
- Provider、Config、Bot、Persona 业务仓储与 selected state 已在第 6 模块确认；本模块只确认 backup data port 如何调用它们。
- QQ/NapCat runtime 已在第 8 模块确认；本模块只确认 QQ login backup 通过 `QqLoginRepositoryPort` 进入 backup data port。

## Resource Center 真源

当前 Resource Center 的数据真源是 Room + feature data store：

- Room 表：`resource_center_items` 与 `config_resource_projections`。
- DAO：`core/db/src/main/java/com/astrbot/android/data/db/resource/ResourceCenterDao.kt`。
- entity：`ResourceCenterItemEntity`、`ConfigResourceProjectionEntity`。
- migration：`DbMigrations.kt` 创建新表、索引，并从旧 config MCP / skill 表 seed 初始资源和 projection。
- data owner：`FeatureResourceCenterRepositoryStore`。
- production port：`FeatureResourceCenterPortAdapter -> ResourceCenterPort`，由 `ResourceRepositoryBindings` 提供。

当前兼容语义：

- 如果新表有 projections，`compatibilitySnapshotForConfig(...)` 使用 Resource Center 表。
- 如果某个 config 还没有 projections，则从旧 `ConfigProfile.mcpServers / skills` 或 config aggregate 投影生成兼容 snapshot。
- `seedFromLegacyConfigTablesIfNeeded(...)` 只在 resource/projection 表为空时从旧 config aggregate seed。
- `dao-owner-allowlist.txt` 当前仍允许 `FeatureResourceCenterRepository.kt` 读取 `ConfigAggregateDao`，原因是 Resource Center 还需要从旧 config MCP / skill 字段投影；这是明确边界债务，不应扩张。

不要把测试里的 `FeatureResourceCenterRepository` / `ResourceCenterRepository` facade 写成 production mainline。

## Backup 主线与覆盖面

当前生产主线：

1. `SettingsBackupViewModel` 调 `SettingsBackupPort`。
2. `SettingsBackupPresentationModule` 将 `AppSettingsBackupPortAdapter` 绑定为 `SettingsBackupPort`。
3. `AppSettingsBackupPortAdapter` 把 settings API DTO 映射到 core backup service DTO。
4. `AppBackupService` 与 `ConversationBackupService` 是 injectable production entry，内部私有持有实例级 repository。
5. `BackupModule` 绑定 `AppBackupDataPort`、`ConversationBackupDataPort` 和 `BackupParticipantRegistry`。
6. `HiltAppBackupDataPort` 从 bot/provider/persona/config/conversation repository store 与 `QqLoginRepositoryPort` 读取 snapshot / restore。
7. App backup restore stage 按 providers、personas、configs、bots、conversations、qqAccounts、ttsAssets 顺序 apply，失败时按已应用 stage 逆序 rollback。
8. Conversation import 当前走 `importSessionsDurable(...)`，full backup 的 conversation restore 也通过 `restoreSessionsDurable(...)`。

当前 `AppBackupModuleKind` / settings backup module 只覆盖：

- `BOTS`
- `PROVIDERS`
- `PERSONAS`
- `CONFIGS`
- `CONVERSATIONS`
- `QQ_ACCOUNTS`
- `TTS_ASSETS`

当前 manifest modules 只写出：

- `bots`
- `providers`
- `personas`
- `configs`
- `conversations`
- `qqLogin`
- `ttsAssets`

`BackupParticipantRegistry` 已把以下模块标为 `SUPPORTED`：

- `bots`
- `providers`
- `personas`
- `configs`
- `conversations`
- `qq-login`
- `tts-assets`

同时把以下模块标为 `PLANNED`：

- `plugins`
- `resources`
- `cron`

因此当前明确未纳入 App backup 的范围包括：

- `resource_center_items`
- `config_resource_projections`
- `cron_jobs`
- `cron_job_execution_records`
- `plugin_config_snapshots`
- `plugin_state_entries`

`configToJson(profile)` 当前已覆盖 `webSearchEnabled`、`proactiveEnabled` 和 `includeScheduledTaskConversationContext`，但仍不能把 Resource Center、Cron 或 plugin config/state 写成 backup 已覆盖。

## Logs 与 Runtime Cache Cleanup

当前日志主线：

- `RuntimeLogStore` 是 UI 可观察 log buffer port。
- `DefaultRuntimeLogStore` 是 ring buffer 实现，默认保留 2,000 条最近日志，按 throttle 刷新 `StateFlow`，并通过 `RuntimeLogSink` 输出。
- `RuntimeLogger` 是轻量 logging fun interface，Hilt 从 `RuntimeLogStore` 构造。
- `RuntimeLogMaintenanceService` 与 `DefaultRuntimeLogMaintenanceService` 管理自动清理设置和 `maybeAutoClear(...)`。
- `RuntimeLoggingModule` 提供 `RuntimeLogStore`、`RuntimeLogger`、`RuntimeLogCleanupSettingsStore` 和 `RuntimeLogMaintenanceService`。
- `RuntimeLogViewModel` 通过注入的 `RuntimeLogStore` 与 `RuntimeLogMaintenanceService` 供 `LogScreen` 读取、复制、清空和设置自动清理。

当前 runtime cache cleanup 主线：

- `RuntimeCacheMaintenancePort` 暴露 `state` 和 `clearSafeRuntimeCaches()`。
- `DefaultRuntimeCacheMaintenanceService` 只清理固定相对路径，且使用 canonical boundary check 防止目标逃逸。
- 当前固定清理目标为 `runtime/downloads`、`runtime/tts-out`、`runtime/rootfs/ubuntu/var/cache/apt/archives`、`runtime/rootfs/ubuntu/root/.config/QQ/nt_qq/global/nt_data/Log`。
- `RuntimeCacheModule` 用 `context.filesDir` 和 `RuntimeLogger` 提供 `RuntimeCacheMaintenancePort`。
- `RuntimeCacheViewModel` 与 `RuntimeCacheCleanupCard` 是 settings hub 的 UI 入口。

旧 `RuntimeLogRepository` / `RuntimeLogCleanupRepository` 当前只在测试兼容 facades 中保留；`RuntimeLogCompatFacadePlacementTest` 明确要求这些 legacy runtime log facades 不出现在 production main source。

## 当前风险与阻塞项

- 本轮未运行 Gradle 或测试，因此不能把本模块状态扩展为构建通过或测试通过。
- `core/backup` 不是完整 backup repository owner；它只是 participant contract / registry。App backup repository 当前仍在 `app/src/main/java/com/astrbot/android/core/db/backup`。
- Resource Center 已是新真源，但 backup 当前仍不覆盖 `resource_center_items` / `config_resource_projections`。
- Backup participant registry 中 `resources`、`cron`、`plugins` 只是 `PLANNED`，不能写成 supported restore。
- Settings presentation 的架构合同禁止直接 import backup/cache/container/plugin/runtime implementation 或 DAO 类型；新增 UI 操作应继续走 injected port / ViewModel。
- Runtime cache cleanup 只能扩展固定安全目标；不能把任意 `filesDir` 或用户输入路径交给 recursive delete。
- `static-repository-usage-allowlist.txt` 当前无生产例外；settings/resource/backup 新代码不得恢复 static repository facade。
- `dao-owner-allowlist.txt` 中 Resource Center 对 `ConfigAggregateDao` 的读取是 legacy projection debt，不应扩张到更多 DAO。
- `global-singleton-allowlist.txt` 仍只有 app string helper 和 QQ stateless mapper；不得为 backup/resource/settings 增加有状态 singleton allowlist。
