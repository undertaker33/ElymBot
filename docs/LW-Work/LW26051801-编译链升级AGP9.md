# LW26051801 编译链升级 AGP9

## 原始请求

用户要求继续执行编译链升级，目标固定为 `AGP 9.2.0 + Gradle 9.4.1`，并同步 KSP 与 API 等级。

## 任务边界

- 本轮范围：Gradle wrapper、AGP、KSP、Kotlin/Compose/Hilt/Room 配套、API 36.1 编译基线、必要的构建合同和 warning-free 修复。
- 非目标：包名重命名、`Astrbot-Android` 命名剔除、业务行为改造、Git 写入。

## 变更文件

- `gradle/wrapper/gradle-wrapper.properties`
- `gradle/wrapper/gradle-wrapper.jar`
- `build.gradle.kts`
- `gradle.properties`
- `build-logic/build.gradle.kts`
- `build-logic/src/main/kotlin/ElymBotKotlinJvmConventionPlugin.kt`
- Android 模块 `build.gradle.kts` 文件：移除 `org.jetbrains.kotlin.android`，改用 AGP 9 built-in Kotlin；Compose 模块保留 `org.jetbrains.kotlin.plugin.compose`。
- Room/Hilt/KSP 相关模块：同步 `Room 2.8.4`、`Hilt 2.59.2`、`KSP 2.3.8`。
- 构建合同和 warning-free 测试修复：`BuildBaselineContractTest.kt`、Cron/Provider/cache/plugin 测试辅助文件。
- 验证任务重新生成了 `artifacts/plugins/**` 下 5 个 sample plugin artifact 文件。

## 实现摘要

- Gradle wrapper 升级到 `9.4.1`，保留 `networkTimeout=60000` 与 distribution URL 校验。
- AGP 升级到 `9.2.0`，根构建统一设置 `compileSdk = 36`、`compileSdkMinor = 1`、`minSdk = 29`、Java 17。
- Android Kotlin 迁移到 AGP 9 built-in Kotlin 路线，删除 Android 模块中的 `org.jetbrains.kotlin.android`。
- KSP 升级到 `2.3.8`，解决 built-in Kotlin 与 KSP generated sources 的兼容问题。
- Room 升级到 `2.8.4`，修复旧 Room compiler 在新 KSP/Kotlin 组合下的 `unexpected jvm signature V`。
- Compose 移除旧 `composeOptions.kotlinCompilerExtensionVersion`，使用 Kotlin Compose compiler plugin。
- 添加 Kotlin compiler 迁移参数，消除 Kotlin 2.2 annotation target 与 data class copy visibility warning。
- 修复测试代码中的 Kotlin 2.2 空值、unchecked cast、参数名和 deprecated API warning。

## 风险与回滚

- AGP 9 built-in Kotlin 是本轮关键迁移点；回滚时需同时回退 AGP/KSP/Kotlin/Compose 配置，不能只降某一个插件。
- `KSP 2.3.8` 与 `Room 2.8.4` 是联动升级；单独降级 Room 可能重新触发 KSP processor 失败。
- sample plugin artifact 文件由验证任务重新生成，提交前应由人工确认是否接受这些二进制/JSON 产物变化。
