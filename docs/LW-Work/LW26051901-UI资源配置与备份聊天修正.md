# LW26051901 UI资源配置与备份聊天修正

## 原始请求

用户提供 6 张截图并要求调整多处 UI：

1. 运行状态浮层删除红框内容展示区；所有状态都不再显示该详情区。百分比进度移动到进度条右侧上方。进度条下方的“正在从网络下载并安装运行时资源”及其他状态说明全部删除。
2. 数据备份页中，“完整备份”模块上移到第一位，其余模块按字母顺序排序。
3. 各个备份详情页顶部内容被顶栏遮挡，需要修正。
4. Chat 页面软键盘弹出时，底部输入框会被遮挡，需要避让。
5. 配置页 MCP、Skill 配置以“我的 - 资源中心 - MCP/Skill”页为唯一真源；配置页改成选择式，参考白名单列表，只能选择资源中心已有资源。管理弹窗可跳转资源中心，返回后回到配置页；配置未保存时退出或跳转需提示是否保存。

用户随后要求：“派子代理做，每个子代理仅装载一个模块的上下文，作为 lw-work 任务。”

## 任务边界与非目标

本次边界：

- 按用户要求以轻量开发记录收口。

- 每个子代理仅装载对应模块上下文，并限制在各自写入范围内。
- 协调者负责合并结果、修正跨模块顶栏返回逻辑、执行验证并写入本记录。

非目标：

- 不改资源中心的资源定义模型和存储真源。
- 不新增正式 Design、Todo、Feedback、Run Log 或 ADR。
- 不更新 `docs/context/`。
- 不执行 Git commit、tag、push、merge 或 PR。
- 不做模拟器截图验收和人工视觉回归。

## 实现摘要

- 运行状态浮层：
  - 移除状态详情展示区。
  - 移除进度条下方状态说明文案。
  - 将百分比显示放到进度条右上方。

- 备份页：
  - `完整备份` 固定第一位。
  - 剩余备份模块按标题排序。
  - 备份详情页统一消费顶栏内边距，避免内容被顶栏遮挡。

- Chat 输入框：
  - 输入栏从单纯 `navigationBarsPadding()` 调整为同时响应 `navigationBars` 和 `ime` inset。
  - 软键盘弹出时底部输入栏随 IME 避让。

- 配置页 MCP / Skill：
  - 配置页不再内联编辑 MCP / Skill 原始资源。
  - MCP / Skill 列表来自资源中心投影。
  - 配置页仅选择资源中心已有项，并维护配置局部的启用状态与 Skill 优先级。
  - 保存时将选择结果映射回现有配置字段，保持运行时兼容。
  - 管理入口跳转资源中心；返回时仍回到配置页。
  - 硬件返回、顶栏返回、跳转资源中心均接入未保存确认。
