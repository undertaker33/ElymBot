## 2. 这轮迁移真正发生了什么

1. 主线仍是 feature-first + Hilt-only；`113dc12` 还新增 `GlobalSingletonAllowlistContractTest`，继续把生产全局单例/手写 DI 的边界写进测试合同。
2. Cron / proactive 能力明显扩展：Active Capability 不再只做 create/delete/list，新增 pause/resume/list runs/update/run now；执行链补上 `CronJobRunNowPort`、`ScheduledMessageDeliveryPort`、`ScheduledTaskIntentGuard`、`ScheduledTaskIntentFallbackResponder`。
3. Runtime context / prompt 继续集中到 core：`ResolvedRuntimeContext` 新增 `scheduledTaskContextWindow`；`PromptAssembler` 会注入 scheduled task metadata、只读最近会话上下文和 web search guidance。
4. Web Search 从 plugin 私有策略迁到 core 统一搜索：当前主线是 `core/runtime/search/*` + `di/hilt/SearchRuntimeModule.kt` + `UnifiedSearchPort`，`WebSearchToolSourceProvider` 只负责把 `web_search` 工具桥接到统一搜索。
5. `a6ffa95` 先引入 API provider + HTML fallback；`b790abce` 又把 HTML fallback 替换成 `LocalMetaSearchFallbackProvider`，本地 fallback 当前走 Sogou / Bing / Baidu Web Lite / DuckDuckGo Lite / Bing News + lite crawler。
6. Provider / Config 增加搜索与主动能力口径：Provider 新增 `ProviderCapability.SEARCH` 和 Tavily / Brave / BoCha / Baidu AI Search 类型；Config 新增 `includeScheduledTaskConversationContext`。
7. QQ runtime 增加 web_search 新闻直送：新闻意图下 `QqStreamingReplyService.deliverNewsSearchResultIfNeeded(...)` 会先把搜索事实发给 QQ，再把 tool result 改成 commentary-only follow-up。
8. App Chat 与 QQ 都接入 scheduled-task fallback responder：如果用户明显在设置提醒且 proactive enabled，而模型没有调用 `create_future_task`，host guard 会尝试创建任务并让模型补一条确认回复。
9. Room 升级到 DB 22：`migration21To22` 给 `config_profiles` 增加 `includeScheduledTaskConversationContext`；App Backup 已能读写该字段，但 Resource Center、Cron、Cron execution records、plugin config/state 仍未纳入 backup manifest。
10. 第 24 期后 QQ data/runtime/presentation 已拆到 `:feature:qq:data`、`:feature:qq:runtime`、`:feature:qq:presentation`；`:feature:qq:impl` 当前为空壳/兼容聚合口径，不再承载 QQ production 主线。
11. 第 25 期后 voiceasset data/presentation 已拆到 `:feature:voiceasset:data`、`:feature:voiceasset:presentation`；voiceasset presentation 只经 api/port 与数据层交互，不依赖 provider runtime implementation。
12. Provider runtime 已创建为 `:feature:provider:runtime`；`ProviderRuntimePort` contract 真源在 `feature/provider/api/src/main/java/com/astrbot/android/feature/provider/api/runtime/ProviderRuntimePort.kt`。
13. Backup port closure 已完成：生产 backup 编排走 Hilt 注入 service / owner port / app-integration backup wiring，不再依赖跨 owner static repository mainline。
14. 第 27 期终态收口已落地：static repository usage allowlist 为 0；global singleton allowlist 为 2 permanent、0 temporary；`:architecture-tests` 与 `build-logic` 已存在，`architectureCheck` 当前包含 `:architecture-tests:test`；`:app` 生产依赖已收窄到 `:app-integration`、`:core:ui` 与 feature presentation。
