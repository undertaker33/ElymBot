# v0.6.x

## 新增

- 项目对外名称统一为 ElymBot，并同步应用名、通知标题、运行时文案、说明文档和帮助版本标识。
- 完成插件能力从 Python 方案向 QuickJS 方案迁移，补齐插件 v2 的启动、消息、生命周期、LLM、工具、分发、注册表和运行时会话链路。
- 新增插件治理快照、失败保护、运行日志总线、活跃运行时状态、安装校验与发布校验。
- 新增本地插件、插件市场、插件详情、插件配置、运行日志、工作区和触发器管理体验。
- 新增下载插件和资产的下载弹窗、下载状态展示与状态栏下载进度通知。
- 新增 Cron 定时任务持久化、表达式解析、调度器、工作器、设置页和 ViewModel。
- 新增 MCP Servers、Skills、主动能力、上下文策略、MCP、Skill、Web Search 等工具源。
- 新增运行时上下文基础设施，包括消息转换、平台适配、Prompt 组装、上下文解析、流式模式判断和系统提示构建。
- 新增 MCP Runtime、Runtime Orchestrator、ScheduledTaskRuntimeExecutor 与 runtime network transport。
- 新增搜索意图识别、新闻/天气搜索策略、相关性评分和 Web Search 工具源策略。
- 新增 Resource Center、资源投影、资源兼容层、资源中心页面和 Cron 执行记录。
- 新增 core 与 feature 包结构，覆盖 Bot、Chat、Config、Cron、Persona、Plugin、Provider、QQ、Resource 等模块。
- 新增 Feature-first 初始化、仓库端口、LLM 运行时端口、App Chat runtime port、发送消息用例和流式消息 sink。

## 修复

- 修复插件无可用更新时的反馈出口不一致问题。
- 修复插件动作反馈中的异常分隔符显示问题。
- 修复运行时通知、宿主通知标题、secret 文件头注释等旧品牌名残留问题。
- 修复应用未申请通知权限导致下载通知不可用的问题。
- 优化插件市场、安装器、目录、治理与配置语义一致性。
- 优化聊天、配置、QQ、OneBot 桥接、Cron、MCP、Web Search 和 App Chat 插件运行时的协作稳定性。
- 优化 Runtime Context、Scheduled Task、HTTP Client、Runtime Network、Plugin LLM Pipeline 与 ChatViewModel 的边界。
- 优化 Resource Center、Cron 编辑、Cron 请求处理、主动能力 facade 和工具源上下文。
- 补充 DB schema、配置映射、插件运行时、Cron、Resource Center、Runtime Context、Web Search、Feature-first 和 ViewModel 测试。
