# v0.7.x

## 新增

- 新增 Cron、Plugin、QQ、Resource 等模块的 feature-first 分层边界。
- 新增核心 ToolSource 合约、工具描述缓存策略、Cron 用例、Cron 端口、主动能力任务端口和 Cron runtime service。
- 新增插件治理、管理、运行时端口、RuntimeFacade、ToolSourceBridge 与 V1 legacy adapter。
- 新增 QQ 会话、平台配置、运行时端口、OneBot 解析、消息运行时、回复发送、媒体映射和 QQ runtime graph。
- 新增 Resource Center 端口、兼容用例和展示控制器。
- 新增 core runtime 分层，将备份、TTS、容器、上下文、LLM、secret、session lock 等能力迁入 core。
- 新增结构化备份仓库、导入规划、备份 manifest、TTS 备份归档、运行时日志和容器桥接服务。
- 新增 App Chat runtime 拆分，包括会话控制、插件命令、预备回复、Provider 调用和运行时服务。
- 新增首轮 Hilt 基础设施、Bootstrapper、Repository Port、Runtime Service 与 ViewModel Dependency 模块。
- 新增数据库、插件供应、运行时服务、ViewModel 依赖、插件运行时、插件宿主能力和运行时网络等 Hilt 模块。
- 新增结构化启动链，包括前置条件、仓库初始化、引用守卫、插件运行时观察和运行时启动。
- 新增严格 Hilt-only 迁移路径和 Feature Repository Port Adapter。

## 修复

- 修复 Manifest、网络安全、Profile 引用、仓库 selected state 与 ViewModel 事务边界中的稳定性问题。
- 修复备份导入兼容、TTS 资源、容器运行时安装、secret 存储、NapCat 反向连接和 Cron 调度中的多处边界问题。
- 优化 App Chat、QQ、Cron、Provider 与插件 Host Capability 的联动。
- 优化插件运行时执行、治理、调度、分发、Bootstrap Host、Tool Loop 和 runtime graph。
- 优化启动链、NapCat root launcher、容器运行时入口、插件目录同步、包校验和 QQ 登录/流式回复。
- 移除旧手工容器、旧 ViewModel helper、legacy repository adapter 与手动 runtime subgraph，降低依赖图复杂度。
- 补充 Hilt、Feature-first、Repository Port、QQ Runtime、Plugin Runtime、Startup Chain、Strict Hilt-only 和 ViewModel 测试。
